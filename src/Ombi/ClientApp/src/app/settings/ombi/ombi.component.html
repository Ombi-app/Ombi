General

Core Ombi application settings including API access, update channel, and global behavior preferences.

View documentation

Application

Configure the base URL and API access for your Ombi instance.

Base URL

Set a base URL if Ombi is hosted behind a reverse proxy at a sub-path (e.g. /ombi). Must start with a forward slash.

API Key

Used for external API access. Click the refresh icon to regenerate.

Update Branch Stable Develop

Controls update checks on the About page only.

Default Language -- {{lang.nativeName}}

Default language for search results. Users can override in their preferences.

Behavior

Control how Ombi handles notifications, requests, and data collection.

Suppress auto-approve notifications

Don't send notifications when a user with auto-approve permission makes a request.

Hide requests from other users

Users will only see their own requests instead of all requests.

Auto-delete available requests

Automatically remove requests once they become available.

Days before deletion

Number of days after a request becomes available before it's automatically deleted.

Anonymous analytics

Help improve Ombi by sharing anonymous usage data like browser type (via Google Analytics).
